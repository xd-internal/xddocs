> Four containers, one cohesive stack.

XplurData is intentionally simple. Every component runs as a Docker container and communicates over an internal bridge network. There are no external dependencies — no managed databases, no cloud services, no API keys.

Images are loaded from the src path relative to docs.html. To add images to any page, add an "image" or "image-grid" section to docs.json — no code changes needed.

### System Diagram

**Sources:** Your App (OTLP SDK) · telemetrygen · Any OTel Agent

**Ingest:** OTel Collector :4318

**Storage:** Doris FE (SQL router) · Doris BE (column store)

**Frontend:** FastAPI REST :8000 · React UI :80


### Network Ports

| Port | Protocol | Used By | Description |
| --- | --- | --- | --- |
| 80 | HTTP | Browser | Web UI |
| 4318 | HTTP | Your applications | OTLP HTTP endpoint |
| 4317 | gRPC | Your applications | OTLP gRPC endpoint |
| 9030 | TCP | Internal | Doris FE SQL port |
| 8000 | HTTP | Internal | FastAPI backend |


:::tip

Only ports 80 and 4318 need to be exposed externally. All other inter-service communication happens on the internal Docker network.

:::
