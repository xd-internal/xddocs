> Common issues and how to resolve them.


### Web UI shows a blank page or loading spinner indefinitely

Doris takes 3–5 minutes to fully initialise on first boot. Wait and refresh. If it persists beyond 10 minutes, check Doris FE logs: docker compose logs doris-fe


### OTLP data not appearing in the UI

Verify the collector is receiving data: docker compose logs otel-collector. Check that your SDK endpoint is http://`<host>`:4318/v1/logs (not 4317 unless using gRPC). Ensure no firewall is blocking port 4318.


### Port 80 or 4318 already in use

Edit docker-compose.yml and change the host port mapping. For example, change "80:80" to "8080:80" and access the UI at port 8080.


### Doris backend fails to start

Doris BE requires at least 4 GB RAM and vm.max_map_count >= 2000000. Run: sudo sysctl -w vm.max_map_count=2000000


### docker compose command not found

You may have Docker Compose v1 installed. Try docker-compose (with hyphen) instead, or upgrade to Docker Desktop which includes Compose v2.


### Cannot connect to the database

Doris FE exposes a MySQL-compatible interface on port 9030. Connect with: mysql -h 127.0.0.1 -P 9030 -u root
